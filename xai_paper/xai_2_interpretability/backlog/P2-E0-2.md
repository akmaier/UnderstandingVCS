### Trajectory tensor (per frame, axis 0 = frame)
`ram`(n,128) · `cpu`(n,7)=[A,X,Y,SP,PC,P,cycles] · `opcode`(n,) (byte at
post-frame PC) · `tia_registers`(n,64) · `tia_derived`(n,10) · `tia_collisions`
(n,8) · `tia_inpt`(n,6) · `riot`(n,11) · `cart`(n,2) · `framebuffer`(n,H,160) ·
`actions`(n,) · `frames`(n,). Column-name lists exported as module constants and
mirrored into the JSON `layout`.

### ENV note for SM Review (not a blocker)
The DoD's "within the conformance horizon" is honoured: recorded **30 frames =
RAM_HORIZON_FRAMES** (`enforce_horizon=True`, `horizon_clamped=false`). The SI
artifact uses `--fast-boot` (60 NOOP + 4 RESET, **no construction probe**)
because the probe-included xitari-parity reference boot (~188 emulated frames)
did not finish in tens of minutes at jaxtari's ~205× slowdown. SI is NOT one of
the probe-sensitive games (only never-re-inited free-running counters like
surround's $7d need the probe), so `--fast-boot` is bit-exact for SI within the
horizon; `meta["reference_boot"]=false` is recorded so a downstream conformance
check is aware. The default `record()` boot remains the full xitari-parity
reference; `--fast-boot` is opt-in. For probe-sensitive games or large batches,
run on the cluster GPU venv (see `cluster.md`). No fabricated results.
