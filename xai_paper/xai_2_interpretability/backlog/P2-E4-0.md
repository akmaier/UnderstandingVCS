## Goal
Stand up the Phase-B pilot: run Integrated Gradients on a single chosen output `y`
(e.g. Pong ball-pixel or score) for one game and compare the attribution map against the
ground-truth intervention oracle (E1-1) and gradient oracle (E1-2). This validates the
attribution harness end-to-end (output selection, baseline choice, scoring against the
oracle) before the full method fan-out in E4-1..E4-13.

## Notes / handoff
See experiment_design.md §5 (Phase B table) and §1 (oracle; content-path gradient caveat —
IG vanishes on index/position outputs, so pick a content output for the pilot). This pilot
fixes the output-selection + oracle-scoring contract reused by all E4 method items.

### DONE (P2 S2) — what shipped
**Substrate pivot, like the E1 oracle:** the pilot is implemented in **Julia**
(`pilot_ig_vs_oracle.jl`) on the verified jutari foundation (jaxtari eager is ~205× slower,
SCRUM §7); `pilot_ig_vs_oracle.py` is a thin dispatcher that preserves the DoD command path
and forwards to the `.jl` via the warm primary jutari project.

**What it does** — Integrated Gradients (Riemann-midpoint, baseline = all-zeros RAM,
completeness err = 0.0) of one Pong output over the RAM tape (Zygote, `soft_ram_peek`
one-hot content read), mapped onto the **exact oracle cause list** (E1-1's RAM/TIA/joystick
causes), then scored on the three §5 faithfulness metrics — all **on the TRUE VCS**:
- corr (Pearson + Spearman) with the oracle's |Δy(u)| causal map;
- deletion/insertion AUC by **re-running the real ROM** with top-IG causes occluded/restored
  (every curve point is a genuine emulator re-run, not a surrogate);
- precision@k vs the true causal top-k.
Plus a **harness positive control**: scoring the oracle's own |Δy| as the candidate map
returns corr=1, precision@k=1 — proving the metrics reward faithfulness (so the IG numbers
are a true measurement of IG, not a metric artefact).

**Run** (jutari, fast):
`julia --project=<primary>/jutari tools/xai_study/phaseB_attribution/pilot_ig_vs_oracle.jl`
(default = faithful headline `p0_score`@f300; `--output ball_pixel` = the §1 contrast;
`--selftest` = the in-script self-check). DoD path also works:
`python tools/xai_study/phaseB_attribution/pilot_ig_vs_oracle.py --game pong --output ball_pixel`.

**Results (jutari, seed 0, f+30 horizon):**
- `p0_score` @f300 (live CONTENT score — IG alive): **corr=0.728, spearman=0.998,
  precision@3=1.0, deletion AUC=0.028 (↓), insertion AUC=0.917 (↑)**; IG top-3 == oracle
  top-3; IG completeness err=0.0. IG is **faithful** here.
- `ball_pixel` @f120 (POSITION/INDEX — the §1 vanishing): **corr=0, precision@3=0**, IG
  max|attr|=0 (no differentiable RAM read for a discrete sprite column) while the oracle has
  real signal — the **'plausible ≠ faithful'** headline; the intervention oracle is the sole
  ground truth there.
- A score output at a **0-score frame** (e.g. `--output p1_score`@f120) shows a *second*,
  distinct IG failure: the **IG baseline-degeneracy** (x − baseline = 0 ⇒ IG = 0) — recorded
  honestly, useful for the paper.

**Artifacts (SPEC §R):** `out/pilotB_faithfulness_ig_pong_p0_score.{json,npz}` (headline) +
`out/pilotB_faithfulness_ig_pong_ball_pixel.{json,npz}` (contrast). JSON carries
{paper,phase,method,game,state,target_output,metric_name,value,n,seed,where,commit,
oracle_ref,timestamp,arrays} + `extra.metrics` + `extra.harness_positive_control` +
per-cause IG/oracle maps; npz carries ig_over_ram(128), per-cause attribution, the oracle
|Δy| vector, and the raw deletion/insertion curves.

**Contract fixed for E4-1..E4-13** (reuse `compute_faithfulness`/`write_faithfulness`):
output selection (content vs position), the IG→oracle-cause mapping, and the
corr / del-ins-AUC-on-the-true-VCS / precision@k scoring against E1-1.

**Caveat:** `state=f300+30` for the headline is inside Pong's long-horizon bit-exactness
(the oracle asserts a byte-identical re-run before scoring); a non-Pong game must re-check
its own conformance horizon.
