## Results (6 core games; jutari HARD, f120+30)
interchange==exact-patch oracle: **TRUE all 6** (max|Δ|=0.0 — harness validated by an
independent fresh replay). **IIA_aligned = 1.0** all 6 (the true oracle alignment realises the
high-level abstraction at the readout — positive control). **IIA_misaligned = 0.0** all 6 (a
wrong cell never reproduces it — negative control discriminates). **alignment_accuracy
(non-transient) = 1.0** all 6 (DAS recovers the true variable alignment exactly on every cell
with a within-horizon causal signal). alignment_accuracy(all, conservative): pong 0.875,
breakout 0.889, space_invaders 0.941, seaquest 0.652, ms_pacman 1.0, qbert 0.95 — every deficit
is fully explained by TRANSIENT cells (1–9/game) the program re-derives within the horizon
(present ≠ used, experiment_design.md §6), recorded not hidden. Artifacts:
`out/das_<game>.{json,npz}` + `out/das_core_summary.json`. Hardware-variable alignment uses
T1/T2 (no T3). Aligning to a NAMED game-concept variable uses the E2-1 candidate concept labels
(T3-candidate) for the variable names only; the causal scoring is T1/T2 (exact oracle).
