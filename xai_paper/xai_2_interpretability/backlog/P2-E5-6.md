of the feature), and monosemanticity. Expected (method matrix §7): **Partial** — recovers

### Done (P2 Sprint 5)
Implemented as **`tools/xai_study/phaseC_mechanistic/sae.jl`** (JULIA, not the tentative
`sae.py`): the causal-use test must RE-RUN the real ROM via the intervention oracle, which
is the jutari substrate (jaxtari eager is ~205× slower — SCRUM §7), and the offline SAE
fit is dependency-free manual GD in Julia (adds no package to the shared env). Generalises
the pilot SAE (`pilot_patch_sae.py`) to all 6 core games + a d_hidden×l1 sweep, and adds
the causal-use ablation the pilot lacked. Cluster-shardable (`--game` / `--shard i
--nshards n --shard-kind game` / `--out-dir` / `--roms-dir`(ignored) / `--where`).

Run (all 6 core games, ~1m46s local):
`julia --project=<repo>/jutari tools/xai_study/phaseC_mechanistic/sae.jl`

**Headline per game** (best of a 3×3 d_hidden{16,32,64}×l1{0.01,0.05,0.2} sweep, 4000
epochs, selected by held-out FVE): match-rate = feature↔variable match (|r|≥0.7 over varying
known T3 cells); ablate==oracle = the bit-exact causal-use positive control; F/S/M = the

**Finding (PARTIAL, as method-matrix §7 predicted):** the SAE RECOVERS features
(match-rate 0.83–1.0 — a learned feature tracks almost every known variable) and
reconstructs well (FVE_held 0.83–0.99 for 5/6; qbert overfits — FVE_train 0.99 but held-out
0.086, a real generalisation-failure data point). BUT faithfulness-to-computation is weak:
F≈0/negative (the SAE surfaces high-variance not necessarily causal cells) and
monosemanticity M is low (0.0–0.27) — ablating a "matched" feature, decoding back to RAM and
re-running the real ROM rarely yields a clean single-variable causal effect (e.g. Pong
feature#62 is the top match for BOTH ball_x and ball_y — polysemantic; target-share 0.33 vs
0.75). "Matched" ≠ "causally used as a clean monosemantic variable" — present, not used.
