DONE (P2 S4). Built `tools/xai_study/phaseA_kording/A3_tuning.jl` — generalizes the
SI pilot's A3 over the 6 core games. Per candidate cell it computes TWO tuning
families: (1) LUMINANCE tuning (hardware, no-T3) = |corr| of the cell's value
time-series vs per-frame mean screen luminance, with the frame clock as the
epiphenomenal negative control; (2) GAME-VARIABLE tuning (T3) = |corr| vs every
candidate-concept signal (the OCAtari/AtariARI labelled cells, self-column
excluded) + a derived score signal; selectivity = max |corr| over game variables.
SCORE = spurious-tuning rate (strongly-tuned cells, selectivity ≥ τ=0.7, that the
intervention ORACLE says are NON-causal) for both families. F/S/M scores the
selectivity-as-importance map against the same oracle causal map (do(0)/do(+17)/
do(+37) whole-screen break), with the oracle-as-method positive control (F==1,
asserted in selftest). Per-game RomSettings parity (CLAUDE.md #95) + a local parity
recorder (avoids jutari_oracle.settings_for's Generic fallback for ms_pacman/qbert/
seaquest) + ms_pacman→mspacman ROM alias.
