## Goal
Build the primary ground-truth oracle: for a VCS output `y` and candidate cause `u`,
intervene (occlude / clamp / replace / resample), re-run deterministically, and record
Δ`y`(u) as the exact causal map. Batched via SOFT-STE (bit-exact to HARD). This is the
reference object every Phase A/B/C method is scored against.

### jutari (Julia) note — the implemented path
Substrate-pivoted to jutari (the proven fast real-ROM path; jaxtari eager is ~205×
slower and stalled inside its first boot). New Julia code, NO emulator core touched:
- `tools/xai_study/common/jutari_oracle.jl` — run helper: load real Pong, xitari-parity
  boot (60 NOOP + 4 RESET), deterministic replay-to-frame, byte-exact snapshot,
  `deepcopy` checkpoint (boot+replay paid once), RAM/TIA/joystick interventions, and a
  dependency-free NumPy `.npy`/`.npz` writer (NPZ isn't in the shared depot).
- `tools/xai_study/ground_truth/oracle_intervene.jl` — the oracle: outputs `y` =
  {p0_score (RAM $0D), p1_score (RAM $0E), ball_pixel (causally-located cell),
  bg_pixel (negative control), n_changed_px (whole-screen intervention magnitude)};
  causes `u` = the candidate RAM bytes from `t3/out/candidates_pong.json` (set base+17 /
  occlude→0) + a TIA colour register + a do(action) paddle input. Default target_frame
  120 / horizon 30 (a *live* frame, so position causes actually move the ball).
- `tools/xai_study/ground_truth/test_oracle.jl` — 8 assertions, all PASS.

Run (≈13 s incl. JIT; ≈16 s for the test — jutari is fast):
