## Goal
Implement occlusion attribution (Zeiler & Fergus 2014) — Δ`y` per occluded region — over
the VCS causes for the chosen output `y`, on the core game set, scored against the §1
oracle. Note: occlusion on the true VCS is itself a coarse intervention oracle, so expect
high faithfulness when the perturbation is a valid intervention (method matrix §7).

## Result (measured, jutari/Julia; target_frame=120 horizon=30 topk=3 seed=0)
Occlusion is INTERVENTION-BASED, so unlike the gradient family it does NOT vanish on the
position/index output (the §7 "Succeed/partial — faithful when the perturbation is a valid
intervention" prediction, in numbers). Per-game corr(occlusion, oracle |Δy|):
- CONTENT path: qbert 0.999, ms_pacman 0.811, breakout 0.750, pong 0.705, space_invaders 0.684,
  seaquest 0.0 (occ L1=0 — the chosen content byte moves under the oracle's set+17 but not under
  occlude→0; honest null at this state). Deletion AUC 0.012–0.12 (steep early drop), high insertion.
- POSITION path (where vanilla saliency/IG identically VANISH): pong 0.837, qbert 0.529,
  space_invaders 0.471 are ALIVE; breakout/seaquest/ms_pacman position cells are occlusion-null
  at this frame (the occlude→0 baseline doesn't disturb that specific sprite cell — honest).
Harness positive control (oracle-as-method) corr=1 / precision@k=1 on every non-degenerate column.
BUDGET per output = 1 intact + N single-cause occlusions + 2·(N+1) del/ins re-runs (N=18–48 causes,
57–147 real-ROM re-runs/output), all continuing from one cached deepcopy checkpoint; recorded in each
§R record under extra.budget. Cluster-shardable: `--shard i --nshards n --shard-kind game` (round-robin
over the 6 core games) + `--out-dir` → tools/cluster/xai_array_jl.sbatch as `--array=0-5%N`.
