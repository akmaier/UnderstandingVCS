## Goal
Turn `tools/xai_study/common/` from specs into a small runnable library: deterministic
game loading, replay-to-state, the self-describing results-record writer (SPEC §R), and
seed management. This is the shared foundation every later experiment imports, so its
schema and APIs must be stable and disjoint from per-experiment runners.

## Definition of Done
- [ ] `tools/xai_study/common/` is importable as a package (loader, replay-to-state, results writer, seeds)
- [ ] round-trips a results record: writes `out/<phase>/<exp>_<game>[_<state>].json` per SPEC §R `{paper, phase, method, game, frame/state, target_output, metric_name, value, ci/stderr, n, seed, where, commit, oracle_ref, timestamp}` and reads it back
- [ ] sibling-array convention (`.npz`) documented and exercised by a unit test
- [ ] unit-tested: runs to completion from `python -m pytest tools/xai_study/common/`
- [ ] nothing outside `file_scope` changed; Paper-1 gates untouched/green
- [ ] committed + pushed to main (rebase-before-push); primary pulled ff-only
- [ ] `status: done`

### Done (P2 S1)
**Built** `tools/xai_study/common/`:
- `loader.py` — `resolve_rom(name)` / `load_game(name)`: resolves a game by name to a
  ROM (curated `xitari/roms/<name>.bin` → sweep-resolved → fuzzy match of the full
  collection, NTSC-preferring, mirroring `resolve_roms.py`) and builds a jaxtari
  `StellaEnvironment` with the same per-game `RomSettings` map as `tools/jaxtari_dump.py`.
  xitari-parity boot by default (60 NOOP + 4 RESET + construction probe). ROMs read in
  place; gitignored files found in the primary checkout via `XAI_PRIMARY_REPO` or the
  `.claude/worktrees/` path heuristic.
- `replay.py` — `to_frame(env, actions, frame)` / `trajectory(env, actions)`:
  deterministic replay to a target frame; `Snapshot(ram, screen, …)`. Enforces the
  Paper-1 conformance horizon (RAM 30 / screen 60 frames) by default; `enforce_horizon=False`
  to go further on purpose.
- `results.py` — `ResultRecord` + `write_record` / `read_record`: the SPEC §R schema
  (`paper,phase,method,game,state,target_output,metric_name,value,stderr,ci,n,seed,where,
  commit,oracle_ref,timestamp,arrays,extra`), one `<exp>_<game>[_<state>].json` per record,
  array payloads in a **sibling `.npz`** named in `arrays`. `commit`/`timestamp` auto-filled;
  NumPy scalars coerced. **Emits per-record JSON only** (no `results_index.csv`).
- `seeds.py` — `seed_everything` / `env_seed` (Python + NumPy + env NOOP RNG from one seed).
- `conftest.py` — puts repo root + `tools/` on the path (import as `tools.xai_study.common`
  or `xai_study.common`); `requires_jaxtari` skip-marker.

**Run command** (DoD: `python -m pytest tools/xai_study/common/`):
```
XAI_PRIMARY_REPO=<primary-checkout> PYTHONPATH=<worktree> \
  <primary>/jaxtari/.venv/bin/python -m pytest tools/xai_study/common/ -v
```
**Result: 7 passed, 1 warning in 1305.54s** (the slow leg is jaxtari's eager boot of
Pong; the JAX `uint64→uint32` warning is benign, emitted inside jaxtari). The 6
pure-Python tests run in 0.12 s; the Pong test loads by name, replays 6 frames
(`ram (128,) uint8`, `screen (210,160)`), asserts deterministic-replay RAM equality
across two fresh envs, and round-trips a real SPEC §R record (+ `.npz` for ram/screen).

**Note on test boot:** the Pong test uses `construction_probe=False` (single boot) to stay
runnable in CI — it exercises the full harness API; the construction probe is a Paper-1
conformance detail, not needed to validate the schema/loader. `load_game(...)` still
defaults `construction_probe=True` for real experiments. No `out/` artifact is committed:
this item's file_scope is the library + tests only, and per-experiment `out/<phase>/…`
records are produced by later items (`out/` is gitignored anyway).

**No blocker.** Nothing outside `file_scope` changed; the emulator core is untouched.
