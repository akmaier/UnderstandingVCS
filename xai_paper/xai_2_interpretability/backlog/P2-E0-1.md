## Goal
Turn `tools/xai_study/common/` from specs into a small runnable library: deterministic
game loading, replay-to-state, the self-describing results-record writer (SPEC §R), and
seed management. This is the shared foundation every later experiment imports, so its
schema and APIs must be stable and disjoint from per-experiment runners.

## Definition of Done
- [ ] `tools/xai_study/common/` is importable as a package (loader, replay-to-state, results writer, seeds)
- [ ] round-trips a results record: writes `out/<phase>/<exp>_<game>[_<state>].json` per SPEC §R `{paper, phase, method, game, frame/state, target_output, metric_name, value, ci/stderr, n, seed, where, commit, oracle_ref, timestamp}` and reads it back
- [ ] sibling-array convention (`.npz`) documented and exercised by a unit test
- [ ] unit-tested: runs to completion from `python -m pytest tools/xai_study/common/`
- [ ] nothing outside `file_scope` changed; Paper-1 gates untouched/green
- [ ] committed + pushed to main (rebase-before-push); primary pulled ff-only
- [ ] `status: done`
