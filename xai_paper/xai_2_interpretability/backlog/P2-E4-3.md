## Notes / handoff
experiment_design.md §5 (row "Guided Backprop") — explicitly calls for the Adebayo et al.
2018 sanity check. Cell/pixel scores against T1 (no T3); object-level needs T3.

**Built** `tools/xai_study/phaseB_attribution/guided_backprop.jl` (Julia), reusing the
IG-pilot scorer + the sibling content/position split. Guided BP = vanilla |∂y/∂u| with the
negative-gradient suppression rule `g⊙1[g>0]`.

**HONEST SUBSTRATE FINDING (DoD-required):** the soft VCS has NO deep ReLU stack — its
nonlinearities are a distance-softmax (relaxed read), straight-through round/clamp
(identity-backward), and a stop-gradient branch. The explained content output is a single
LINEAR one-hot read (∂y/∂u = e_idx ≥ 0), so the guided suppression removes NOTHING ⇒ **Guided
BP degenerates to (rectified) vanilla saliency** (`guided_equals_vanilla=true`, Δcorr=0 on all
6 games). The suppression rule IS genuinely wired (verified: `guided_demo_rectifies` provably
zeros the negative entries of a mixed-sign surrogate).

**Headline faithfulness (content path, vs the §1 oracle):**

Position output (`ball_pixel`): Guided BP **VANISHES** on all 6 (max|attr|=0, corr=0) — the §1
"plausible ≠ faithful" contrast. Harness positive control (oracle-as-method): **corr=1.0,
precision@3=1.0** on every game (the scorer rewards faithfulness).

**Adebayo et al. 2018 sanity check — FAILS on every game (the expected, informative result).**
Scrambling the ROM boots the program to a totally different RAM-at-frame state (~95–127/128
bytes change) yet the Guided-BP / saliency map is **byte-identical** (mean cosine to the real
map = 1.0). Diagnosed `failure_mode = model_invariant_saliency_constant_jacobian`: the linear
read's Jacobian e_idx is a CONSTANT one-hot independent of RAM, so the saliency is **provably
model-invariant** — it points at the read index whatever the program computes. The intervention
oracle (re-runs the real program) does NOT fail this check; only the gradient saliency does.
