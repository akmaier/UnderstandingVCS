### DONE (P2 S5) — what was built
Written in **Julia** (SCRUM §5/§7 — experiments run on JUTARI locally), like the IG sibling
(E4-5), not the `.py` placeholder named in `file_scope`. The runner is
`tools/xai_study/phaseB_attribution/expected_gradients.jl`; artifacts are
`tools/xai_study/phaseB_attribution/out/expected_gradients_*`. It reuses the IG baseline-sweep
sibling (`ig_baseline_sweep.jl`, P2-E4-5) wholesale — the per-game ROM-alias/RomSettings map,
bit-exact assertion, oracle |Δy|, deletion/insertion AUC on the TRUE VCS, content/position
readers, `pick_content_idx` (so EG scores the SAME content byte as IG), and the IG primitive
for the head-to-head — plus the IG-pilot scorer (pearson/spearman/precision@k, oracle-as-method
positive control). Reference distribution D = genuine per-frame RAM states (frames 1..target,
the E0-2 recorder semantics, built on the validated per-game env).

Method: Expected Gradients (Erion et al. 2021, NMI) — `EG_i(x)=E_{x'~D,α~U(0,1)}[(x_i−x'_i)·
∂y/∂x_i|_{x'+α(x−x')}]`, Monte-Carlo over M=64 real-state references, R=5 stability seeds.
Run: `julia --project=<repo>/jutari tools/xai_study/phaseB_attribution/expected_gradients.jl
--games core` (self-check: `--selftest`). 6 core games × 2 outputs (content + ball_pixel),
12/12 self-checks pass, 0 errors; harness oracle-as-method control corr=1/p@1=1 everywhere.

**Headline finding (content path), EG corr vs single-baseline IG(zeros) corr, Δcorr:**
- pong   ram[54] EG 0.407 = IG 0.407 (Δ≈0) — EG ALIVE (byte varies over D); stability cos=1.0
- breakout ram[30] EG 0.000 vs IG 0.750 (Δ=−0.75) — EG DEGENERATE (byte=192 const over D)
- ms_pacman ram[16] EG 0.000 vs IG 0.811 (Δ=−0.81) — EG DEGENERATE (byte=98 const over D)
- qbert  ram[39] EG 0.000 vs IG 0.999 (Δ=−1.00) — EG DEGENERATE (byte=255 near-const, 1/120)
- space_invaders ram[81] EG 0.000 vs IG 0.215 (Δ=−0.21) — EG DEGENERATE (byte=246 const)
- seaquest ram[118] EG 0.000 = IG 0.000 — both 0 (byte=0 const; degenerate either way)
- position (ball_pixel): EG max|attr|=0 for ALL games/seeds → near chance (plausible ≠
  faithful; averaging real baselines cannot manufacture a missing gradient — same as IG §1).

**Result (sharper than the textbook "EG removes IG's baseline sensitivity"):** on the one-hot
content path ∂y/∂u=e_idx is CONSTANT, so when EG IS alive it scores IDENTICALLY to IG (Pong:
Δcorr≈1e-16, stability cosine=1.0, corr_std≈0 — provably stable, no faithfulness gain) at M×
the compute. But EG inherits — and on the VCS AMPLIFIES — the baseline degeneracy: EG_self=
(x_self−x'_self), and on an ON-DISTRIBUTION reference window the concept byte is (near-)constant
across all references (5/6 games), so (x−x')=0 at the only live cell ⇒ EG VANISHES where a
synthetic `zeros` baseline (0≠byte) keeps IG alive. So on-distribution references make the EG
degeneracy MORE likely, not less. Flagged per game via `eg_self_baseline_invariant` +
`n_refs_self_differs` + `content_byte_strictly_constant_over_D`; stability (mean pairwise cosine
of per-cause attribution + corr spread/std over the seed ensemble) is the §R "stability"
deliverable. The intervention oracle is the sole reliable attribution where EG degenerates.
