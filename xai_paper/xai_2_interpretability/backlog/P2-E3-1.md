## Notes / handoff
experiment_design.md §4 A1. Ground truth T1/T2 (no T3). Built on the validated Phase-A
foundation (pilot_si.jl template + jutari_oracle.jl + oracle_intervene.jl).

**Method.** Recover the inter-cell DATA-FLOW GRAPH over the per-game candidate cells
(`candidates_<game>.json`) via pairwise interventions: the *true* read/write graph is the
EXACT intervention oracle's cell→cell adjacency (edge i→j iff the exhaustive do-set on i —
occlude→0, set→base+17, set→base+37, full horizon — changes cell j; bit-exact re-run, so
each edge is a real causal effect, not a world-model guess). The *recovered* graph (the
classical connectomics method under test) is a thin single-shot recovery: one perturbation
do(i:=base+17), 1-step latency. Scored P/R/F1 + graph-edit-distance (symmetric off-diagonal
edge difference fp+fn) + normalised GED. Positive control = oracle-as-method (full do-set +
full horizon) which scores F1=1, GED=0 on every game — the harness rewards a faithful graph.
F=F1, S=generalisation to a HELD-OUT base+37 true edge set, M=1−over-claim(false edges).

**Results (f30+30 NOOP, jutari, bit-exact on all 6).** Mean recovered F1 = 0.41; positive
control F1 = 1.0 / GED = 0 on all games. Per game (nodes, true_edges, rec_edges, P/R/F1, GED):
pong (8, 6, 0) 0/0/0 GED 6; breakout (9, 1, 1) 1/1/1 GED 0; space_invaders (17, 9, 3)
0.67/0.22/0.33 GED 8; seaquest (23, 11, 2) 0.5/0.09/0.15 GED 11; ms_pacman (19, 18, 0)
0/0/0 GED 18; qbert (20, 1, 1) 1/1/1 GED 0. The quantified-Kording lesson: the thin
single-shot/short-latency recovery badly under-recovers long-latency, sign-dependent
data-flow (pong/ms_pacman recall 0; seaquest 0.09); breakout/qbert score 1.0 only because
their data-flow at this state is a single edge (trivially matched). seaquest has no jutari
RomSettings yet → booted Generic (recorded in each record's `extra.settings`); all 6 are
bit-exact at this state/horizon so every Δ is a clean causal effect.

**Artifacts.** `out/A1_connectomics.{json,npz}` (combined: per_game_PRF_GED_FSM (6,7)) +
`out/A1_<game>.{json,npz}` per game (true/recovered/heldout/oracle-method adjacency UInt8
matrices, edge lists, P/R/F1/GED, F/S/M).
