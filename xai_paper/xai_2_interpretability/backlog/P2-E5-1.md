**DONE (Sprint 5).** Built on JUTARI (Julia), generalising the Phase-C pilot
(pilot_patch_sae.jl) from Pong to all 6 core games. The `file_scope` listed `.py`
paths, but per SCRUM §7 + the task contract all P2 experiments run on jutari (the
fast real-ROM bit-exact path; jaxtari eager is ~205× slower), and the pilot it
generalises is `.jl` — so the implementation is
`phaseC_mechanistic/activation_patching.jl` (sibling of the pilot), writing to the
declared `out/activation_patching_*` dir.

- **Run:** `julia --project=<repo>/jutari tools/xai_study/phaseC_mechanistic/activation_patching.jl --games core`
  (self-check: `--selftest`). XAI_PRIMARY_REPO resolves the gitignored ROMs.
- **Method:** for each candidate state site, two patch families — DONOR (value from
  a genuinely different LEFT/RIGHT joystick-context run) and DIRECTED (synthetic
  base+17, mirroring the oracle `set` cause) + a content-path TIA patch. Each patch
  is a single-site write at frame t + a real re-run; the EXACT intervention oracle Δ
  is the same write via an INDEPENDENT fresh-from-boot replay → recovered==exact.
- **Result (all 6 core games):** recovered == exact-patch oracle
  (max|recovered−exact| = 0.0) and data-flow firing-pattern **P = R = 1.0** — the
  §7 "Succeed (causal by construction)" prediction, exact on the VCS. Honest
  "present ≠ used" audit: most one-frame RAM patches are transient/clobbered
  (a-priori target recall 0.11–0.29), a minority genuinely used (e.g. qbert ram@39
  moves 5772 screen px).
- **Artifacts:** `out/activation_patching_<game>.{json,npz}` (×6, §R schema; npz =
  recovered/exact (patches×outputs) + y_baseline + |rec−exact|) +
  `out/activation_patching_core_summary.json`. JSON+NPZ validated numpy-loadable.
- **Self-check PASS:** bit-exact ✓, recovered==exact ✓, P=R=1.0 ✓, directed
  self-cell patch Δ=+17 ✓.
