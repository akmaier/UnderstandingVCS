## Goal
Package the P2 tasks, the ground-truth oracle, and the F/S/M metrics into a reusable benchmark so a
third party can run a single interpretability method end-to-end and obtain a comparable score. Must
handle ROM non-redistribution cleanly (hashes / AutoROM pointers, never committing ROM bytes) and
expose a documented entry point that consumes a method's attribution/circuit output and emits the
scored record (SPEC §R).

## Definition of Done
- [ ] runs to completion from this command: `python -m tools.xai_study.benchmark.run --method <example> --game pong --self-test`
- [ ] a method can be plugged in and scored end-to-end against the oracle, output written in the schema (SPEC §R)
- [ ] ROMs are referenced by hash/AutoROM, never redistributed (SCRUM §7: use in place, never commit)
- [ ] nothing outside `file_scope` changed; Paper-1 gates untouched/green
- [ ] committed + pushed to main (rebase-before-push); primary pulled ff-only
