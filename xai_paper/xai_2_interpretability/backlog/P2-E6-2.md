## Goal
Package the P2 tasks, the ground-truth oracle, and the F/S/M metrics into a reusable benchmark so a
third party can run a single interpretability method end-to-end and obtain a comparable score. Must
handle ROM non-redistribution cleanly (hashes / AutoROM pointers, never committing ROM bytes) and
expose a documented entry point that consumes a method's attribution/circuit output and emits the
scored record (SPEC §R).

- [ ] ROMs are referenced by hash/AutoROM, never redistributed (SCRUM §7: use in place, never commit)

## Done (P2 S6)
Reusable benchmark packaged at **`tools/xai_study/compare/benchmark/`** (the Sprint-6 directive
placed it under `compare/`; the SPEC's tentative `tools/xai_study/benchmark/**` was reconciled to
this co-located path so it sits beside the E6-1 leaderboard it feeds — no other item touches it).
Four pieces + a runnable example, all PASS:

- **TASK set** (`tasks.py`): 6 core games (game_set.json §G) × 3 regimes {content, position,
  ball_pixel}. **14 scorable tasks** ship; 4 (game,regime) pairs excluded as **degenerate**
  (constant/all-zero oracle column within the conformance horizon → no signal): breakout/ball_pixel,
  breakout/position, seaquest/ball_pixel, seaquest/position (flagged in manifest.json, not hidden).
- **ORACLE interface** (`oracle.py`): `load_oracle(game,regime) -> OracleMap{cause_names, abs_delta,
  top_k(k), scorable}` — the exact §1 `|Δy(u)|` map, read from the committed Phase-B §R records
  (`extra.cause_names` + `extra.oracle_abs_delta_per_cause`). **No ROM needed to score.**
- **METRIC API** (`metrics.py`): `score_method(out, oracle, k=3, rerun_fn=None)` → pearson/spearman
  corr-with-oracle, precision@k, deletion/insertion AUC (live, optional ROM hook), F/S/M triad,
  FSM_composite. Formulas mirror the committed Julia scorer `pilot_ig_vs_oracle.jl`. F =
  `max(0,corr)` clipped to [0,1], comparable to the E6-1 leaderboard.
- **EXAMPLE + entry point** (`example_method.py` + `run.py`): plug-in contract
  `method(task,oracle)->{attribution,S,M}`; bundled oracle_copy (positive control, F==1), uniform
  (floor, corr 0), random, magnitude_proxy (oracle-free toy). Records → `out/<method>_<game>_<regime>.json`
  in SPEC §R with `extra.triad.F`.
- **ROM non-redistribution**: `rom_manifest.json` references the 6 core ROMs by real SHA-256 +
  AutoROM name only (never ROM bytes); SCRUM §7.

**RUN (sync) + self-check PASS (5/5):**
`python -m tools.xai_study.compare.benchmark.run --self-test` → oracle_copy F==1 on all 14 tasks;
uniform at floor (corr 0); magnitude_proxy finite F=0.271 on pong/content; oracle interface
corr==1 vs itself; ≥6 tasks. Demonstration run wrote 14 magnitude_proxy records (mean F=0.143).
Leaderboard (E6-1) re-run unchanged (no regression). DoD command form
`run --method oracle_copy --game pong --self-test` works; module + direct-script + python-API
entry paths all verified.
