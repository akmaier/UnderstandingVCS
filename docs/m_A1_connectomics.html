Two adjacency matrices over the candidate RAM cells: the true data-flow graph and the graph this method recovered. A bright cell (row = cause, column = effect) is an edge; the difference between the two panels is the error. dataflow_graph_F1_vs_oracle = 0.000 — this example only (pong, state f30+30); the audit aggregate is below.

What it does

Reconstructs the inter-cell data-flow graph by perturbing each RAM cell and recording which other cells change — the program's 'connectome'. Scored by graph F1 against the true read/write graph from the disassembly.

How it's scored

F1 of the recovered data-flow graph against the true read/write graph from the disassembly (an edge = perturbing one RAM cell changes another). 0.0 — single-shot perturbation recovers none of the true edges.

The score is measured against the §1 intervention oracle — never against another interpretability method. F (faithful) is always vs the oracle; see the execution stack.
