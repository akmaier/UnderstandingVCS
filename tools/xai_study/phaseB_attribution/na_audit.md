## Why this is a write-up, not a 6-game method run

Every other Phase-B item runs a method over the 6 core games and scores its
attribution map against the §1 intervention oracle with F/S/M plus an
oracle-as-method positive control (template: `pilot_ig_vs_oracle.jl` /
`phaseA_kording/pilot_si.jl`). **This item cannot**, because the methods it
covers have nothing in the VCS to attach to. The subject of Paper 2 is the
**Atari VCS itself** — a deterministic 6502 CPU + TIA + RIOT executing a fixed
ROM (no neural network; no learned policy; *agents are P5, not P2*). The methods
below each require a neural-network architecture (convolutional feature maps,
attention matrices, learned channels) or a *learned policy* to extract. None of
those ingredients exists on the bare VCS, so there is **no operand** to run the
method on and **nothing to score**. Recording this honestly *scopes* Phase B and
supplies the "architecture-specific methods don't apply" row of the leaderboard
(E6) — the measured narrowness of popular XAI.

## Per-method audit

### Grad-CAM — *does not apply*

- **Family:** CNN class-activation mapping. **Citation:** Selvaraju et al. 2017 (ICCV).
- **Requires (missing ingredient):** convolutional feature maps (a last conv layer whose channels are spatially pooled and weighted by class-gradient).
- **Present in the VCS?** **No.**
- **Why it does not apply:** Grad-CAM forms a class-discriminative heatmap by weighting the activation maps of a CNN's final convolutional layer by the gradient of the class logit w.r.t. those maps, then ReLU-ing the weighted sum. The VCS has NO convolutional layers, NO learned channels, and NO class logits: it is a 6502 CPU + TIA + RIOT executing a fixed ROM. There is no last-conv-layer activation tensor to weight, so the construction has no operand.
- **Why a naive re-targeting onto VCS state is *not* this method:** Projecting register/RAM/colour-clock state into a 'feature map' grid and weighting by ∂y/∂state is NOT Grad-CAM: those are raw causal state cells, not learned conv channels, and the resulting map is just the content-path gradient (already covered by vanilla-gradient / Grad×Input, E4-1/E4-2). The channel-pooling + ReLU that DEFINE Grad-CAM are vacuous without conv channels.

### Grad-CAM++ — *does not apply*

- **Family:** CNN class-activation mapping. **Citation:** Chattopadhyay et al. 2018 (WACV).
- **Requires (missing ingredient):** convolutional feature maps + higher-order (2nd/3rd) gradients of the class score w.r.t. those maps.
- **Present in the VCS?** **No.**
- **Why it does not apply:** Grad-CAM++ refines Grad-CAM's per-channel weights with pixel-wise higher-order gradient terms over the SAME final conv feature maps. It inherits Grad-CAM's hard requirement for conv channels; the VCS has none, so the refinement has nothing to refine.
- **Why a naive re-targeting onto VCS state is *not* this method:** Higher-order gradients of a VCS pixel w.r.t. state cells exist (the substrate is differentiable, content path), but computing them is not Grad-CAM++ — without conv feature maps the per-channel weighting that the formula sums over is undefined. The result would again be a (higher-order) saliency map, not a CAM.

### Attention rollout — *does not apply*

- **Family:** Transformer attention attribution. **Citation:** Abnar & Zuidema 2020 (ACL).
- **Requires (missing ingredient):** self-attention matrices across transformer layers (per-head token-to-token attention weights to multiply through, with the residual/identity mix).
- **Present in the VCS?** **No.**
- **Why it does not apply:** Attention rollout recursively multiplies the layer-wise self-attention matrices (averaged over heads, mixed with the residual identity) to trace how input tokens influence a later representation. The VCS is not a transformer: there are NO attention heads, NO token sequence, and NO attention matrices to multiply. The method's entire input — the stack of A^(l) matrices — does not exist.
- **Why a naive re-targeting onto VCS state is *not* this method:** One could build a token-to-token influence matrix from the VCS's true data-flow (which RAM cell reads/writes which), but that is the §1 intervention/data-flow oracle (or Phase-C path patching), NOT attention rollout: there is no learned, softmax-normalised attention being rolled up. Calling the data-flow graph 'attention' would mislabel ground truth as a method under test.

### VIPER (policy distillation to a decision tree) — *does not apply*

- **Family:** Policy extraction / surrogate. **Citation:** Bastani et al. 2018 (NeurIPS).
- **Requires (missing ingredient):** a LEARNED policy π(a|s) (a trained agent / Q-network) to distil into an interpretable decision tree via DAgger-style imitation.
- **Present in the VCS?** **No.**
- **Why it does not apply:** VIPER extracts a verifiable decision-tree policy by imitating a trained neural policy (Q-network) over visited states, using the Q-values to weight the imitation. Paper 2's subject is the PROGRAM (the VCS as a deterministic input→output function), not an agent: there is NO learned policy, NO action distribution to imitate, and NO Q-function to weight by. The thing VIPER distils does not exist in this study. (Agents are P5, not P2.)
- **Why a naive re-targeting onto VCS state is *not* this method:** Fitting a decision tree to (state → next-RAM / pixel) of the VCS itself is not VIPER: VIPER's object is a LEARNED control policy, and its guarantee is about imitating that policy's returns. The VCS's own state-transition is the exact program — a tree over it is a (lossy) re-description of known ground truth, not a policy-distillation explanation. Faithful recovery of the program is Phase C (circuit discovery), scored vs the true data-flow.

### CAM (class activation mapping) — *does not apply*

- **Family:** CNN class-activation mapping. **Citation:** Zhou et al. 2016 (CVPR).
- **Requires (missing ingredient):** a global-average-pooled final conv layer feeding a single linear classification layer (the GAP+FC architecture CAM is defined on).
- **Present in the VCS?** **No.**
- **Why it does not apply:** Vanilla CAM requires the specific CNN topology of a GAP'd last conv layer wired to one FC classifier; the heatmap is the conv maps weighted by that FC's class weights. The VCS has neither conv maps nor a learned classifier head, so the construction is undefined.
- **Why a naive re-targeting onto VCS state is *not* this method:** There is no GAP+FC head to read class weights from; any 'importance grid' over VCS state is the content-path gradient, already covered by E4-1/E4-2.

### Feature visualisation / CAV — *does not apply*

- **Family:** Learned-channel interpretation. **Citation:** Olah et al. 2017 (Distill); Kim et al. 2018 (TCAV, ICML).
- **Requires (missing ingredient):** LEARNED hidden channels / directions in a network's activation space to optimise an input toward, or to probe with concept-activation vectors.
- **Present in the VCS?** **No.**
- **Why it does not apply:** Feature visualisation synthesises inputs that maximally activate a chosen LEARNED unit/channel/direction; TCAV measures sensitivity to a learned concept-activation vector in hidden space. Both require a trained network with hidden channels that have emergent semantics. The VCS's state cells are NOT learned channels — they are hardware registers/RAM with KNOWN, fixed semantics (T1/T2). There is nothing learned to visualise or to define a CAV in.
- **Why a naive re-targeting onto VCS state is *not* this method:** Optimising a ROM/input to maximise a pixel is the §1 gradient/IG path (content) or an adversarial-style intervention, not feature-vis of a learned channel. Probing a KNOWN register's role is the §2/Phase-C probing task (with control tasks), where the labels are ground truth, not discovered concepts.

## What *does* apply (the contrast, scored elsewhere in E4)

- **Vanilla gradient / saliency (Simonyan et al. 2014)** — applies; scored vs the oracle in E4-1.
- **Integrated Gradients (Sundararajan et al. 2017; P8)** — applies; scored vs the oracle in E4-0 / E4-5.
- **Occlusion on the true VCS (Zeiler & Fergus 2014)** — applies; scored vs the oracle in E4-7.
- **On-distribution counterfactual states (Olson 2021; Atrey 2020)** — applies; scored vs the oracle in E4-12.

These bind to the VCS's true causes and are evaluated as methods under test
(`experiment_design.md` §5). The headline expectation (§7) holds: *causal,
gradient, and mechanistic methods pass; correlational and architecture-specific
methods fail or do not apply.* This audit nails down the "do not apply" half.
